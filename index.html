I spent this week working on two small games. The plan I made with Nimit was to make known games like Tetris and Space Invaders but while browsing through JS games on Monday I came up with a few ideas. I’ve been very excited about being able to turn my ideas into something not only visual but also interactive. Another step visually is to draw out the icons/characters but that’s going to take time and will not help my coding knowledge. If I need a break from Sublime I might try. I have begun to review the Crafty JS game engine which so far seems to have a good balance of help and getting things done on my own. I hope this will help me learn the gaming basics in case I end up doing something like this for a career. THe goal of learning Crafty is to bring into existence an idea that David has. At first I thought it would be impossible but I already have a very basic idea of how I could do it: 2d array, let’s say 10x10 for now. An X and an O on opposite sides, like field[5][0] and field[5][9]. Using Ajax to grab the info, players can type up instructions and have the program run it. ‘Move Right’ would move the X or O ship to the right, adding to the second index in the field set. X’s position would change to field[5][1]. After iteration through the field, images will be added where the ships are, and maybe space-themed tiles elsewhere. Rocks or something to block movement/attacks and somehow have a function check if that space is clear prior to movement. Attacking would work the same way. ‘Fire Up’ checks the first index in field and if the content contains X or O, a ship, that sips loses health. O can Fire Up upon [4][9]. This will be easier to do without tracking projectiles, just show another image of the ship lighting up as if it were shooting something bright.
